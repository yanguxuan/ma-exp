# IL+RL 混合电梯调度算法说明

## 一、算法原理

### 1.1 核心思想

**IL+RL（Imitation Learning + Reinforcement Learning）** 是一种两阶段混合学习策略，旨在结合专家示范的快速启动能力与强化学习的自主优化能力，在电梯调度任务中超越传统启发式算法。

核心逻辑：
> **第一阶段**：通过行为克隆（Behavioral Cloning）模仿专家策略（如 ETA-based），快速获得一个合理的初始调度模型。  
> **第二阶段**：利用深度 Q 网络（DQN）在真实环境奖励下对模型进行微调，使其超越专家水平，适应复杂、随机的客流模式。

### 1.2 两个阶段详解

| 阶段 | 方法 | 输入 | 输出 | 目标 |
|:---|:---|:---|:---|:---|
| **模仿学习** | 监督学习（分类） | 状态特征向量 → 专家动作（目标楼层） | 神经网络策略 | 克隆专家决策，提供热启动 |
| **强化学习** | DQN（双网络+经验回放） | 状态特征向量 + 奖励信号 | 更新的策略 | 优化长期累积奖励，超越专家 |

### 1.3 与经典论文方法的对应

- **Tan (1993)**：提出多智能体可通过共享经验加速学习，本算法使用了“从专家学习”的思想（类似 Tan 的 Case 2 中“从专家猎人学习”）。
- **Crites & Barto (1995)**：首次在电梯调度中应用 RL，使用神经网络 Q 函数。本算法继承其深度 Q 网络框架，但增加了行为克隆预热。
- **Cao et al. (2022)**：尝试 DQN 直接优化电梯，但因环境非 MDP 失败。本文通过模仿学习提供稳定初始策略，缓解了收敛问题，取得了成功。
- **Vaartjes & Francois-Lavet (2025)**：使用 Dueling Double DQN 和自定义动作空间。本算法的 DQN 微调部分采用了类似的架构思想（单一神经网络输出各目标楼层的 Q 值）。

## 二、状态表示与特征设计

### 2.1 状态空间

本算法使用 **21 维** 特征向量作为神经网络的输入，既紧凑又包含足够信息，避免了高维特征导致的学习困难。

| 特征组 | 包含信息 | 维度 |
|:---|:---|:---|
| **电梯状态** | 每台电梯：当前楼层（归一化）、方向（-1/0/1 映射）、负载率、是否移动 | `3 × 4 = 12` |
| **等待人数** | 每层楼当前等待乘客数量（归一化） | `9` |
| **时间特征** | 无（实验表明不加反而更稳定） | `0` |
| **合计** | | **21** |

### 2.2 特征预处理

- **楼层归一化**：`floor_norm = floor / (num_floors - 1)` → 范围 [0,1]。
- **方向编码**：`dir_norm = (direction + 1) / 2` → {-1,0,1} 变为 {0, 0.5, 1}。
- **负载率**：`load_rate = current_load / capacity` → [0,1]。
- **等待人数归一化**：`cnt / max(cnt, 1)` → [0,1]。

### 2.3 为什么没有包含最大等待时间？

实验中发现，加入“最大等待时间”特征（22 维）后，训练不稳定，最终性能反而下降（见第二次实验结果）。分析原因：

- 该特征噪声大，且与“各楼层等待人数”存在共线性。
- 21 维特征已足够，在**回退规则（方案一）** 的辅助下，模型能够隐式学习到长时间等待乘客的优先级。

## 三、模仿学习（行为克隆）细节

### 3.1 专家策略

选择 **ETA-based 算法** 作为专家。该算法已在工业界广泛验证（Otis Compass®, KONE Destination Control），具有良好的性能基准。

### 3.2 数据收集

- 在模拟器中运行专家策略 **10 个 episode**，每个 episode 模拟 1 小时（3600 秒）。
- 在每一步，记录 `(21维状态特征向量, 专家选择的动作)`。动作为目标楼层（0~8）。
- 共收集约 **35,000 条** 样本。

### 3.3 网络结构与训练

| 参数 | 值 |
|:---|:---|
| 输入层 | 21 个神经元 |
| 隐藏层1 | 128 个神经元 + ReLU |
| 隐藏层2 | 128 个神经元 + ReLU |
| 输出层 | 9 个神经元（对应 9 个目标楼层） |
| 损失函数 | 交叉熵（CrossEntropyLoss） |
| 优化器 | Adam，学习率 0.001 |
| 训练轮数 | 10 epochs |
| 批量大小 | 128 |

### 3.4 训练结果

行为克隆损失从约 1.55 降至 0.99，表明网络成功学会了专家的基本调度模式（例如“空车时去等待人数多的楼层”、“载客时送最早乘客”）。

## 四、强化学习微调（DQN）细节

### 4.1 DQN 架构

- **与行为克隆相同的网络结构**（保持输入输出维度一致）。
- **目标网络**：每 10 步更新一次，稳定训练。
- **经验回放**：容量 10000，每次随机采样 64 条经验。
- **探索策略**：ε-greedy，ε 从 0.8 指数衰减至 0.05，衰减系数 0.99。

### 4.2 奖励函数

使用模拟器内置的奖励（与 ETA-based 算法评估时相同）：

| 事件 | 奖励值 |
|:---|:---|
| 成功送达一位乘客 | `+10 + 行程距离` |
| 乘客登梯 | `+1.0` |
| 每步每个等待乘客 | `-0.15` |
| 乘客放弃等待 | `-8 - 2×距离` |

**未使用额外奖励塑形**，让模型自主探索最优策略。

### 4.3 训练轮数

- **30 个 episode**（每个 episode 1 小时）。
- 总时间步约：30 × 3600 = 108,000 秒（模拟时间）。
- 训练耗时约 5-10 分钟（CPU）。

### 4.4 训练稳定性

虽然单 episode 奖励存在波动（从负值逐渐转为正值），但总体呈上升趋势。最终模型在测试中实现了 **零放弃率** 和 **比 ETA-based 更低的平均等待时间**。

## 五、推理与回退规则（方案一）

### 5.1 推理流程

在最终部署（或在实验评估）时，采用以下决策逻辑：

```python
def select_actions(state):
    for each elevator:
        if elevator has passengers:
            go to first passenger's destination
        else:
            # 模型预测最佳目标楼层
            best_floor = argmax(Q_network(features))
            # 回退规则：如果预测楼层无等待乘客，改为等待人数最多的楼层
            if waiting[best_floor] == 0 and any waiting:
                best_floor = argmax(waiting_counts)
            actions.append(best_floor)
```

### 5.2 回退规则的作用

该规则作为 **安全网**，确保模型即使在错误预测时也不会完全空跑。它显著降低了放弃率，是第一次实验成功的重要因素之一。

### 5.3 为什么不直接使用回退规则训练？

回退规则是**启发式**的，会干扰模型自主学习。我们仅在推理阶段应用它，而训练时不使用，让模型自由探索更优策略。

## 六、实验结果

### 6.1 与 ETA-based 基线对比

| 指标 | ETA-based | IL+RL（本算法） | 改进 |
|:---|:---|:---|:---|
| 平均等待时间（AWT） | 35.1 秒 | **30.5 秒** | ↓ 13.1% |
| 平均行程时间（ART） | 146.0 秒 | 156.4 秒 | ↑ 7.1% |
| 放弃率 | 1.9% | **0.0%** | ↓ 100% |
| 吞吐量（人/分钟） | 3.20 | **3.23** | ↑ 0.9% |

### 6.2 关键观察

- **放弃率为零**：模型学会了优先服务等待时间长的乘客，避免超时惩罚。
- **平均等待时间显著降低**：得益于 RL 微调，模型发现了比 ETA 更灵活的调度策略（例如提前预判趋势楼层）。
- **行程时间略有增加**：这是公平调度与节能之间的常见权衡——优先服务长时间等待的乘客可能导致更远的绕行，但总体体验更公平。

## 七、算法优势与局限性

### 7.1 优势

- **无需大量手工特征工程**：端到端学习，适应不同建筑布局。
- **可超越专家策略**：RL 微调能发现启发式算法未考虑的优化。
- **稳定性高**：行为克隆提供了良好的初始策略，避免 RL 冷启动。
- **零放弃率**：兼顾公平性，提升用户体验。

### 7.2 局限性

- **训练仍需仿真环境**：与真实世界的 gap 可能影响部署。
- **泛化能力有限**：改变楼层数量、电梯数量等需重新训练。
- **行程时间略有增加**：在追求低等待时间时，可能牺牲部分行程效率。

### 7.3 与相关论文的关系

| 论文 | 方法 | 与本算法对比 |
|:---|:---|:---|
| Crites & Barto 1995 | 多智能体 Q-learning + 神经网络 | 本算法使用单智能体、模仿学习预热，训练更高效 |
| Cao et al. 2022 | 纯 DQN（失败） | 本算法通过行为克隆解决非平稳问题，成功收敛 |
| Vaartjes et al. 2025 | Dueling Double DQN + infra‑steps | 本算法采用更简单的 DQN，但仍达到可比性能 |

## 八、代码与使用

### 8.1 文件结构

```
src/algorithms/ilrl_based.py      # 推理算法（加载训练好的模型）
train_ilrl.py                     # 离线训练脚本（行为克隆 + DQN）
experiment_runner.py              # 实验运行器（支持加载本算法）
```

### 8.2 训练模型

```bash
python train_ilrl.py
```

训练结束后生成 `ilrl_model.pth`。

### 8.3 在实验中评估

```python
from src.algorithms.ilrl_based import ILRLBasedAlgorithm

runner = ExperimentRunner(config)
runner.add_eta_based()  # 基线
runner.add_algorithm(ILRLBasedAlgorithm(
    num_elevators=3,
    num_floors=9,
    model_path="ilrl_model.pth"
))
runner.run_all()
```
